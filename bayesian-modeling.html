01 — Overview

Project Overview

Bayesian inference treats probability as a measure of belief, not just frequency. Instead of a single point estimate, it produces a full posterior distribution — capturing what we know, what we don't, and how both change as evidence accumulates.

This project implements the full Bayesian pipeline from scratch: conjugate prior-posterior updates for tractable cases, and a Metropolis-Hastings MCMC sampler for cases where the posterior has no closed form. All implementations are validated against PyMC to confirm correctness.

02 — Core Concepts

Three Pillars of This Project

📐
Conjugate Priors
Beta-Binomial, Normal-Normal, and Gamma-Poisson conjugate pairs — exact posterior updates with no sampling required.
🔗
Metropolis-Hastings
Custom MCMC sampler that approximates any posterior by constructing a Markov chain whose stationary distribution is the target.
📊
Prior Sensitivity
Systematic analysis of how strongly-informative vs weakly-informative priors affect posterior conclusions at different sample sizes.
03 — Mathematical Background

Mathematical Foundation

# Bayes' Theorem — the engine of everything
p(θ | x) = p(x | θ) · p(θ) / p(x)
posterior ∝ likelihood × prior

# Beta-Binomial conjugate update
Prior: θ ~ Beta(α, β)
Likelihood: x | θ ~ Binomial(n, θ)
Posterior: θ | x ~ Beta(α + x, β + n - x)

# Metropolis-Hastings acceptance ratio
A(θ*, θ) = min(1, p(θ*|x) · q(θ|θ*) )
                  ( p(θ|x) · q(θ*|θ) )
04 — Implementation

Implementation Approach

  1. Conjugate updates — Implement Beta-Binomial and Normal-Normal updating rules analytically. Visualize how posterior narrows as data accumulates.

  2. MCMC sampler — Build Metropolis-Hastings from scratch with tunable proposal distribution (Gaussian random walk). Track acceptance rate and chain mixing.

  3. Convergence diagnostics — Implement trace plots, autocorrelation functions, and R-hat statistic to verify chain convergence.

  4. Prior sensitivity analysis — Run same model with Uniform, weakly informative, and strongly informative priors across varying sample sizes.

  5. PyMC validation — Compare posterior means, credible intervals, and full distributions against PyMC's NUTS sampler output.

import numpy as np from scipy import stats def metropolis_hastings(log_posterior, theta_init, n_samples, proposal_sd=0.5): """ Metropolis-Hastings MCMC sampler. log_posterior: function returning log p(θ|x) """ samples = np.zeros((n_samples, len(theta_init))) theta = theta_init.copy() accepted = 0 for i in range(n_samples): # Propose new θ from Gaussian random walk theta_star = theta + np.random.normal(0, proposal_sd, size=len(theta)) # Log acceptance ratio (symmetric proposal → simplifies) log_alpha = log_posterior(theta_star) - log_posterior(theta) # Accept or reject if np.log(np.random.uniform()) < log_alpha: theta = theta_star accepted += 1 samples[i] = theta acceptance_rate = accepted / n_samples print(f"Acceptance rate: {acceptance_rate:.2%}") return samples def beta_binomial_posterior(alpha_prior, beta_prior, n_successes, n_trials): """Exact conjugate update for coin-flip model.""" alpha_post = alpha_prior + n_successes beta_post = beta_prior + (n_trials - n_successes) return stats.beta(alpha_post, beta_post)

The Metropolis-Hastings sampler converged to the correct posterior for a logistic regression model with two parameters, matching PyMC's NUTS sampler posterior means within 1% and 95% credible intervals within 0.8 percentage points.

Prior sensitivity analysis showed that with fewer than 20 observations, a strongly informative prior dominates. By 100 observations, even a strongly misspecified prior is overwhelmed by the likelihood — the data takes over.

06 — Insights

Key Insights

  • The posterior is a compromise between prior belief and data evidence. With enough data, the likelihood always wins — Bayesian and frequentist answers converge.
  • MCMC doesn't sample from the posterior directly — it constructs a Markov chain that happens to have the posterior as its stationary distribution. The distinction matters for understanding burn-in and mixing.
  • Proposal distribution width is critical in Metropolis-Hastings. Too narrow → slow exploration (high autocorrelation). Too wide → almost always rejected. Targeting ~23% acceptance is theoretically optimal for high-dimensional posteriors.
  • Credible intervals have a natural interpretation: "there is a 95% probability the parameter lies here, given the data and prior." Confidence intervals carry no such statement.

Future Improvements
